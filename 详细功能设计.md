# 详细功能设计

## 1. 设备数据采集模块

### 1.1 功能描述

通过边缘网关采集传感器数据，并上传至云端存储。

### 1.2 数据类型

| 数据类型 | 字段名 | 单位 | 采样频率 |
|----------|--------|------|----------|
| 振动X轴 | vibration_x | g | 1000Hz |
| 振动Y轴 | vibration_y | g | 1000Hz |
| 振动Z轴 | vibration_z | g | 1000Hz |
| 温度 | temperature | ℃ | 1Hz |
| 电流 | current | A | 10Hz |
| 电压 | voltage | V | 10Hz |
| 转速 | rpm | rpm | 1Hz |

### 1.3 数据预处理

- **去噪**: 低通滤波器去除高频噪声
- **降采样**: 原始数据 1000Hz → 存储 100Hz
- **特征提取**: 
  - 均值、方差、最大值、最小值
  - RMS（均方根）
  - 峰值因子
  - 频域特征（FFT）

### 1.4 边缘网关功能

| 功能 | 说明 |
|------|------|
| 本地缓存 | 网络断开时缓存 7 天数据 |
| 断点续传 | 网络恢复后自动补传 |
| 边缘计算 | 本地预处理，降低云端压力 |
| 远程管理 | OTA 升级、远程配置 |

## 2. 故障预测模块

### 2.1 预测目标

| 预测类型 | 说明 | 预测时长 |
|----------|------|----------|
| 剩余使用寿命 (RUL) | 还能运行多久 | 0-30 天 |
| 故障概率 | 当前状态故障可能性 | 24 小时内 |
| 故障类型 | 可能发生的故障类型 | 提前预警 |

### 2.2 故障类型库

| 故障类型 | 典型特征 |
|----------|----------|
| 轴承磨损 | 振动增大、温度升高 |
| 电机过热 | 电流增大、温度升高 |
| 润滑不足 | 振动异常、噪音增大 |
| 皮带松动 | 转速波动、振动变化 |
| 齿轮箱故障 | 特定频率振动峰值 |

### 2.3 模型输入特征

```
输入特征:
├── 时域特征 (12个)
│   ├── 均值、标准差、峰峰值
│   ├── 偏度、峭度
│   ├── RMS、峰值因子
│   └── 脉冲因子
├── 频域特征 (8个)
│   ├── 主频率、能量占比
│   └── 频谱重心、频谱方差
└── 统计特征 (4个)
    └── 温度趋势、电流趋势...
```

### 2.4 模型输出

```json
{
  "device_id": "motor_001",
  "timestamp": "2026-03-14T10:00:00",
  "health_score": 85,  // 健康评分 0-100
  "rul": 15,           // 剩余寿命(天)
  "failure_probability": 0.15,  // 故障概率 0-1
  "failure_type": "轴承磨损",
  "confidence": 0.82,  // 置信度
  "recommendation": "建议本周安排检查"
}
```

## 3. 报警推送模块

### 3.1 报警等级

| 等级 | 阈值 | 推送方式 | 响应时间 |
|------|------|----------|----------|
| 🟢 正常 | 健康分 > 70 | 不推送 | - |
| 🟡 提醒 | 健康分 50-70 | 微信/钉钉 | 24 小时内 |
| 🟠 警告 | 健康分 30-50 | 微信/钉钉 + 短信 | 4 小时内 |
| 🔴 紧急 | 健康分 < 30 | 电话 + 短信 | 立即 |

### 3.2 报警内容模板

```
【设备报警】
设备: 空压机 #1
等级: ⚠️ 警告
问题: 轴承磨损风险
详情: 健康分降至 45，预测 3 天后可能故障
建议: 建议本周安排更换轴承
时间: 2026-03-14 10:00
```

### 3.3 推送渠道

| 渠道 | 适用场景 | 成本 |
|------|----------|------|
| 微信企业号 | 日常提醒 | 免费 |
| 钉钉机器人 | 日常提醒 | 免费 |
| 短信 | 紧急报警 | 0.1元/条 |
| 电话 | 最高紧急 | 0.5元/分钟 |
