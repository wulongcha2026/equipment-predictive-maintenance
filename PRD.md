# 设备预测性维护系统 - PRD (产品需求文档)

## 1. 项目概述

| 项目 | 内容 |
|------|------|
| **项目名称** | 设备预测性维护系统 |
| **产品定位** | 帮助工厂通过 AI 预测设备故障，减少非计划停机 |
| **目标客户** | 中小型工厂 |
| **核心价值** | 提前预警 → 减少停机 → 节省维修成本 |

---

## 2. 目标用户

- 工厂设备管理人员
- 维修工程师
- 工厂老板（看报表）

---

## 3. 核心功能 (MVP)

### 3.1 设备数据采集

- 支持接入振动传感器、温度传感器、电流传感器
- 通过边缘网关采集数据，上传到云端
- 支持 Modbus、OPC UA 等工业协议

### 3.2 故障预测

- 基于时序数据分析设备健康状态
- 预测未来 1-7 天可能发生的故障
- 给出故障概率和置信度

### 3.3 报警推送

- 微信/短信/钉钉及时通知
- 支持多级报警（提醒 → 警告 → 紧急）

### 3.4 可视化看板

- 设备健康状态一目了然
- 历史故障记录查询
- 设备运行趋势图表

### 3.5 维保工单

- 自动生成维保建议
- 创建和派发维保工单
- 跟踪维保进度

---

## 4. 产品架构

```
┌─────────────┐     ┌─────────────┐     ┌─────────────┐
│  传感器层   │ →   │  边缘计算层 │ →   │   云平台    │
│ 振动/温度/  │     │  数据采集   │     │  数据存储   │
│ 电流传感器  │     │  边缘网关   │     │  AI 预测    │
└─────────────┘     └─────────────┘     │  可视化    │
                                          │  报警推送  │
                                          └─────────────┘
```

---

## 5. 技术选型

| 模块 | 技术方案 |
|------|----------|
| 边缘网关 | 树莓派 / Intel NUC |
| 通信协议 | MQTT / Modbus |
| 时序数据库 | InfluxDB |
| AI 模型 | Python + Scikit-learn / LightGBM |
| 后端 | Python FastAPI |
| 前端 | Vue.js / React |
| 报警 | 微信企业号 / 钉钉机器人 |

---

## 6. 商业模式

| 模式 | 价格 |
|------|------|
| 硬件 + 软件 | 按设备台数收费，首年 2000-5000 元/台 |
| SaaS 订阅 | 300-800 元/台/年 |
| 定制化项目 | 按项目报价，5-20 万 |

---

## 7. 实施步骤

1. **第 1 周**：选择试点设备（如空压机），安装传感器
2. **第 2-4 周**：数据采集 + 清洗 + 初步分析
3. **第 5-8 周**：训练 AI 预测模型
4. **第 9-10 周**：开发可视化看板和报警功能
5. **第 11-12 周**：试点上线，优化迭代

---

## 8. 风险与对策

| 风险 | 对策 |
|------|------|
| 数据量不够 | 先用模拟数据训练，再逐步替换为真实数据 |
| 工厂不配合 | 从老板痛点切入，展示 ROI |
| 传感器故障 | 选用工业级传感器，备用方案 |

---

## 9. 成功指标

- 预测准确率 > 80%
- 非计划停机减少 30%
- 客户满意度 > 90%
